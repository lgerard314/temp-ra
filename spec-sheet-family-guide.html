00 · The bet

The Spec Sheet family.

A working project binder, made public. Documentary chrome at conversational volume — file numbers, sheet revisions, exhibit tags, signed lines. The chrome itself is the trust mechanism.

Thesis

B is the backbone. A's body sizing + manila ground + italic Newsreader pull-quotes are storm-claim modules. C's "OPEN FILE" stamp + first-pass strikethrough are opt-in emphasis devices. One brand, three volumes — same grammar everywhere.

The rules — how the system protects itself from drift

01 · Paper grounds

Bond `#FAFAF6` is the page. Card `#FFFFFF` only as elevated surfaces. Manila `#F0EEE6` only on storm-claim leaves. Pure white is never the page itself.

02 · One accent

Orange `#FB6A1D` is the only chromatic color that ever moves. Status, buttons, exhibit tags, rev numbers, eyebrow labels. Used as punctuation, not as fill.

03 · Mono is data

JetBrains Mono only carries metadata: file numbers, dates, locations, dimensions, square footage, carrier names, status. Never body copy. Never headlines.

04 · Display is loud

Archivo Narrow is uppercase, tight leading, no italic. Body is Work Sans, sentence case. Newsreader italic only inside .ra-quote for testimonials.

05 · Sharp edges

Border-radius is `0` everywhere except photo-meta tags (`2px`). The warranty stamp is the one circle on the entire site.

06 · One break per composition

Every filebar has one cell highlighted in orange (the active one). Every page has one stamp, one pull-quote, one eyebrow. Don't double up.

07 · Real metadata

Every file number, address, carrier name, square-footage on a page must be a real one. Placeholder Latin breaks the system instantly.

08 · Audit before you create

Grep `--ra-` and `.ra-` before adding any token or class. The system is small; new primitives need ≥3 consumers or a one-off comment.

01 · Color

Six tokens. One moves.

Three paper grounds, one accent, two ink tones. Status colors are utility-only. White is never promoted to a brand color.

03b · Motion + frame

Quiet motion. Sharp frames.

Three durations, one curve. Two shadow values (most pages use neither). Four border colors. Five photo ratios. The system prefers print-flat.

03b.1 · Motion — live demos (auto-loop)

--ra-d-fast 120ms

Micro-interaction. Button hover, link color shift, status pill tick. Should feel instant — register but never be noticed.

--ra-d-base 240ms
● ACTIVE · 042

UI state change. Header scroll-collapse, filebar cell hover, photo zoom. The default motion duration site-wide.

--ra-d-slow 480ms
ON FILE

Reveal. Stamp fade-in on hero, ledger row stagger, page transition. Used sparingly — once or twice per page.

Curve: all motion uses --ra-ease — one cubic-bezier site-wide. --ra-ease-in / --ra-ease-out exist as escape hatches but should not appear in normal use.

03b.2 · Header scroll-collapse — interactive

● 14-MIN DISPATCH · DFW + CLE · STORM SEASON OPEN
RA · CONTRACTING (469) 555-0199

Header demonstrates the scroll-collapse pattern. Topbar height tweens to 0, header padding tightens, a 1px --ra-shadow-lift appears on bottom edge. Total motion: --ra-d-base with --ra-ease.

03b.3 · Shadows — narrow allowed list

Default position: no shadow. The system is print-flat. Borders and ground-color shifts (--ra-mist) carry the depth that shadow carries elsewhere. Two shadow tokens exist; if you find yourself adding a third, stop and use a border instead.

--ra-shadow-none

Default. Cards, photos, buttons, ledger rows, hero, CTA banner — none of these get shadow.

--ra-shadow-lift

Allowed only on: sticky header (after scroll), modal/dialog, floating tooltip. Three uses site-wide. Nothing else.

03b.4 · Rule (border) colors

--ra-rule-default

Hairline on paper. Filebar cells, ledger rows, dividers, card borders.

--ra-rule-strong

Slate ink, full strength. Section heading underline, hero frame, CTA outline.

--ra-rule-active

Orange, 2px. The single highlight rule — one cell per filebar, active nav, current step.

--ra-rule-dim

White-at-16% on slate. Footer columns, dark-bar dividers, hero filebar lines.

03b.5 · Photo ratios — locked

Five aspect ratios. Every photo on the site picks one. Crop the photo to the ratio — never crop the ratio to the photo.

21 : 9Hero · cinema
--ra-ratio-cinema21 / 9

Hero photo, full-bleed lede band. Maximum impact, used 1× per page.

16 : 9Section · wide
--ra-ratio-wide16 / 9

Section photos, before/after sliders, video wells. The default landscape ratio.

4 : 3Process · ladder
--ra-ratio-landscape4 / 3

Process step photos, exhibits, paired with body copy. Print-page feel.

3 : 4Crew · portrait
--ra-ratio-portrait3 / 4

PM portraits, vertical detail shots. The only vertical ratio.

1 : 1Thumb · gallery
--ra-ratio-square1 / 1

Project thumbnails, gallery grids, small cards. Never on hero.

03b.6 · Hover language — hover any tile

Three moves, used singly or in combination. Never shadow on hover — shadow is reserved for sticky-header / modal / tooltip only.

Border shift Hairline → 2px orange (`--ra-rule-active`). The strongest move. Used for active item, focus, primary hover.
2px lift `translateY(-2px)` over `--ra-d-fast`. Subtle motion that reads as motion. Buttons, ledger rows, photo frames.
Ground shift `--ra-bond` → `--ra-mist`. Depth without shadow. Cards, tiles, list rows.
Combined 2px lift + orange border + ground shift. Reserved for primary CTA cards (project tiles, file cards).
